# Desarrollo

Al iniciar el proyecto, fui definiendo las características y como iba a ser el mismo utilizando la **metodología SCRUM** con la aplicación **Jira**, creando un backlog con todas las historias de usuario necesarias y las estimaciones hechas. Una vez teniendo esto fui definiendo los sprints a medida que iba completando las tareas, revisando en cada iteración que puntos podía mejorar para agilizar el desarrollo.

![Backlog](/images/projects/PersonalWebPage/personal-web-page-img-2.jpg)

La utilización de un tablero ** Kanban** para organizar los estados de las tareas que debía realizar por sprint fue de gran utilidad, el tener de manera visual las tareas y su estado me permitió no perder tiempo en pensar “y ahora que debería hacer?” ya que todo estaba ya organizado, agilizando asi notoriamente tiempo de desarrollo

![Tablero Kanban](/images/projects/PersonalWebPage/personal-web-page-img-3.jpg)

# Tecnologías

<Tecnologies>React,Next,Typescript,Styled-Components</Tecnologies>

Elegi utilizar **React** porque es la librería con la que me siento mas cómodo trabajando, en conjunto con **Next.js** permite crear aplicaciones que se renderizen del lado del servidor, asi como generar paginas web estáticas (como son las de los proyectos), característica que permite mejorar el SEO y posicionamiento en los buscadores.

La elección de **Styled-Components** es porque quería darle una estética personalizada a mi pagina web, cosa que esta librería me permite hacer de una manera sencilla.

Y el uso de **TypeScript** ya es una practica mas que adoptada en mi desarrollo día a día, mejorando la comodidad al programar y evitando posibles errores.
